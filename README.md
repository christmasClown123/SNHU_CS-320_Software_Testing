# SNHU_CS-320_Software_Testing

Functional and secure code starts with a greater understanding of the restrctions and limitations of the code and it's purpose. After assessing, you start the process of drafting and developing the code. I prefer to work in building blocks to allow myself to track new concepts or organize my thoughts. The goal is a working prototype that facilitates the, previously mentioned, restrictions and limitations. Testing becomes the critical part to ensure that the code handles normal operations, exceptions, and data appropriately. The user will get you most of the way with how their end product should look like, it is up to the programming to close the distance. Experience teaches the best lessons, so it becomes critical to bring up any thing you've seen that they may not be thinking about when explaining their application. Leading questions are another wonderful tool to allow you to specifically define the finer requirements.
